Ask Kurasu

General entry point — the user doesn't need to know which feature fits.

User
Asks a general question, in their own words — optionally with location.
Orchestrator
Minimal required fields — reaches ready almost immediately.
Specialist
Decides: answer directly / ask exactly one clarifying question / recommend a specific existing feature.
↳ if answering directly
Search
Google Search, preferring official/government sources for rules and procedures.
Response
Direct answer, or a one-line recommendation to open the right feature.
🏥

Clinic Finder

Matches symptoms to nearby care that can actually treat them.

User
Describes symptoms, with location (device GPS or a landmark).
Orchestrator
Collects symptoms + location before handoff.
Search
Finds real nearby clinics/hospitals matching the described symptoms.
Specialist
Explains fit in plain English — what each place actually treats.
Response
Ranked clinic list with map links.
🍽️

Restaurant Guide

Finds where to eat for a craving or dietary need.

User
Describes cravings or dietary needs, with location.
Orchestrator
Collects craving/diet + location.
Search
Finds real nearby restaurants and menu information.
Specialist
Translates/explains menu items and dietary fit.
Response
Restaurant picks with why each one fits.
🏷️

Ingredient Checker

Reads a label photo and checks it against a dietary need.

User
Uploads a product/ingredient photo, states the concern (halal, vegan, allergy…).
Orchestrator
Collects photo + dietary concern.
Specialist
Reads the photo directly (vision) and reasons about each ingredient against the stated concern.
Response
Safe / not safe / unclear verdict, with the specific ingredient(s) responsible.
📦

Delivery Scheduler

The only feature that submits a real request on the user's behalf.

User
Uploads a notice photo/QR or types a tracking number, states a time preference.
Orchestrator
Collects tracking info + time (and optional date) preference.
Deterministic
QR code pixel-decoded directly (OpenCV) — never guessed from the photo.
Specialist
A decoded link alone is enough reason to attempt automation — no OCR branding confirmation required.
Automation
Real headless browser: opens the link → advances to the date/time step → selects time + date → submits.
Response
Genuine confirmation if scheduled — otherwise honest fallback with manual steps.
🆘

Disaster Help

Safety-critical — minimal friction, government data only.

User
Describes an emergency (or just needs a safe place), with location.
Orchestrator
GPS-priority, never delays on crisis type.
Deterministic
Haversine nearest-shelter calc over Japan Gov (GSI) open shelter data.
Specialist
Calm, urgent tone — safety instruction first, then shelters.
Response
Nearest outdoor sites + indoor shelters, each with a map link and source cited.
🗑️

Waste Guide

Rules are set per municipality, not nationally — location matters.

User
Asks a waste question, with location (or names a ward/city directly).
Orchestrator
Location optional — skipped for general questions.
Deterministic
GPS reverse-geocoded to an exact ward/city (OpenStreetMap Nominatim).
Specialist
Searches that exact municipality's real rules — categories, days, bag requirements.
↳ if nothing reliable is found
Fallback
Says so honestly, offers general guidance, suggests Form Decoder & Filler for a local pamphlet.
Response
Concrete sorting rules for that exact address.
📝

Form Decoder & Filler

The most involved flow — a real back-and-forth interview, then two generated bilingual images. Branches into two modes after intent is known.

User
Uploads 1–5 form photos (one per page), states decode or fill.
Orchestrator
Collects photo(s) + explicit intent.
BRANCH A — DECODE
Specialist
One-shot: explains the form's purpose and every field, in plain English.
Response
Plain-English explanation — no file generated.
BRANCH B — FILL
Deterministic
Photo read once: every field, label, choice-option, and branching condition identified.
↻ REPEATS ONE FIELD AT A TIME
Specialist
Asks the next applicable field — silently skips any field whose condition isn't met.
Deterministic
Once complete: answers structured with fill-type (write/circle/check/shade) and position.
Generation
Every page × language pair rendered concurrently — AI image-edit, with a deterministic overlay fallback per page.
Response
Completed form, as real images, in English and Japanese.
